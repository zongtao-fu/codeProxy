# 演进记录（Evolution）

本文件用于记录“结构性变更 / 关键决策 / 规范调整”，帮助后续追溯“为什么现在是这样”。

## 记录原则

- 只记录“会影响后续维护与协作”的变更：目录结构、依赖方向、工具链切换、关键架构决策、重大行为变更。
- 每条记录必须包含：时间、变更内容、原因（why）、影响范围、回滚要点（如适用）。

## 条目模板（可复制）

```
## YYYY-MM-DD · 标题

### 背景

### 选项（可选）
- 方案 A：
- 方案 B：

### 结论

### 影响范围

### 回滚策略（如适用）
```
