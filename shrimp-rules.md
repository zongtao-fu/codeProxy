# 开发守则（code-proxy-admin · 硬性约束）

## 适用范围

- 本规则适用于本仓库。
- 只写“硬性约束”（违反会导致质量/一致性显著下降）；通用规范请看 `rules/*`。

## 工程与依赖（强制）

- **禁止无关改动**：不引入与任务无关的重构/格式化大改/依赖膨胀。
- **依赖变更必须可追溯**：新增/升级依赖要说明动机、影响范围、替代方案，并完成最小验证闭环（见下方）。
- **最小验证闭环（至少）**：
  - Lint：`bun run lint`
  - 格式化：`bun run format`
  - 测试：暂无（当前仓库未配置测试脚本/用例）
  - 构建：`bun run build`

## 栈特定约束（自动识别 · 强制）

- 包管理器：固定使用 `bun`；禁止提交/使用 `npm`/`pnpm`/`yarn` 锁文件。
- 样式：检测到 Tailwind CSS；新增全局样式需说明原因，优先复用工具类/组件库。
- Skill：本项目存在 `.agents/skills/*`；涉及其主题的任务必须优先使用对应 Skill。

## 安全与隐私（强制）

- 禁止把密钥、Token、私有链接、用户数据写入仓库与日志；必要时用占位符并说明。
- 处理文件路径/命令执行时必须防止路径穿越与命令注入（尤其在后端/脚本中）。

## 路径变更联动（强制）

- 任何“项目路径变更”（新增/移动/重命名目录或文件、调整关键导出入口、更新别名路径等）必须同步更新：
  - `AGENTS.md`（索引/任务映射/关键路径）
  - 按需更新 `rules/project-structure.md`、`README.md` 与 `docs/evolution.md`

## AI 决策优先级

- **先遵守本文件**，再遵守 `AGENTS.md` 与 `rules/*`。
- 出现冲突时，选择更严格的限制。
